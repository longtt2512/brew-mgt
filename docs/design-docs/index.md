# Design Docs

Durable design notes — the "why" behind how `brew-mgt` is built. Distinct from
`docs/decisions/` (which records discrete accepted decisions) and from
`docs/product/` (which records what the product does). Design docs hold the
reasoning, beliefs, and shapes that span many decisions.

- [`core-beliefs.md`](./core-beliefs.md) — the principles agents should not
  violate without an explicit decision.

Add focused design docs here as the system grows (e.g. `brew-json-mapping.md`,
`error-model.md`, `async-upgrade-jobs.md`).
